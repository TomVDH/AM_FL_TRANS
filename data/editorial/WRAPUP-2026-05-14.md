## 1. What This Was

*Asses.Masses* is an animated Belgian indie game about a donkey uprising. The dialogue is dense, character-driven, and politically irreverent. The localisation target was not generic Standard Dutch — it was **Flemish tussentaal**: a living register somewhere between formal ABN and pure dialect, full of ge/gij pronouns, Flemish contractions, loanwords, and the specific rhythm of spoken Belgian Dutch.

That specificity is the whole point. A Standard Dutch localisation would have been competent. A Flemish one is the game.

The project ran in two overlapping phases:

1. **Tooling phase** — building the infrastructure to read from and write back to the live Google Sheets corpus (Patrick's working document), with dry-run safety, round-trip verification, and a character codex.

2. **Editorial phase** — walking every cell of every episode against a locked canon of ~700 editorial rules, catching drift, applying corrections, and pushing to remote with verification at each step.

Both phases were necessary. The tooling without the editorial would have been a build with no ship. The editorial without the tooling would have been a spreadsheet nightmare.

## 3. What Was Built

### 3.1 The Pipeline

The corpus lives on Google Sheets (Patrick's working document). We never edited it directly. The pipeline is:

```
Google Sheets (remote, source of truth)
      ↓ pull-snapshot.py
Local .xlsx mirrors (excels/)
      ↓ editorial pass
Corrected local .xlsx
      ↓ push-file.py (dry-run first)
      → diff shown per cell, user confirms
      ↓ push-file.py --apply
Google Sheets updated (tinted green)
      ↓ pull-snapshot.py (re-pull)
      ↓ push-file.py (dry-run again)
      → expect 0 diffs
Round-trip verified ✅
```

Every single correction in the project went through this loop. No cell was edited on the remote directly. No push without a dry-run. No dry-run without a full per-cell readout.

### 3.2 Core Scripts

| Script | What it does |
|---|---|
| `pull-snapshot.py` | Pulls all 11 xlsx files from Google Sheets into a local snapshot directory |
| `push-file.py` | Diffs local xlsx against remote; shows per-cell changes; pushes on `--apply` |
| `push-tab.py` | Same as push-file but targets a single sheet tab |
| `apply-corrections.py` | Applies a correction JSON batch to local xlsx before push |
| `comprehensive-audit.py` | Full corpus scan — flags drift, register violations, terminology mismatches |
| `walk-sheet.py` | Per-sheet cell walker, EN + NL side by side |
| `e{N}_sweep_scan.py` | Per-episode targeted sweep scripts (one per episode, E3–E10) |
| `propose-fixes-e{N}.py` | Generates correction candidates for a given episode |
| `ultimate-check.py` | Final state verification across all episodes against canon |
| `three-way-diff.py` | Compares original baseline → Patrick's version → our version per cell |

### 3.3 The Codex (`codex_verified.json` v3.5)

37 character entries. Each entry carries:

- **pronounsAllowed / pronounsForbidden** — the ge/gij vs je/jij split
- **contractionsAllowed / contractionsForbidden** — `'k`, `'t`, `'n`, etc.
- **dialectalMarkersAllowed** — godverdomme, gulle, etc.
- **articleRule** — `een` vs `ne/nen`
- **negationRule** — superseded corpus-wide by §2 (everyone uses `niet`)
- **registerExceptions** — when a je/jij character shifts to u/uw (formal/Gods/elder)
- **verbalTics** — character-specific speech markers (`*boe-hoe-hoe*`, `, Kameraad`, `EZELS EERST`, etc.)
- **inboundAddressRules** — how others address this character (Kick Ass calls Sick Ass `Snotje`)
- **dutch / dutchShort** — canonical long and short name in Dutch

The codex is machine-readable and was used by sweep scripts to catch register violations programmatically.

## 4. The Editorial System

### 4.1 How corrections worked

Every correction was staged as a JSON file before touching any xlsx:

```json
{
  "id": "E2-014",
  "file": "2_asses.masses_E2Proxy.xlsx",
  "sheet": "E2_World_A2_localization",
  "cell": "J7",
  "speaker": "Sign",
  "english": "NEW Government-Subsidised Highway...",
  "current_nl": "NIEW Overheids-gesubsidiëerde snelweg: brengt stedelijke en landelijke regios...",
  "proposed_nl": "NIEUW Overheids-gesubsidieerde snelweg: brengt stedelijke en landelijke regio's...",
  "rationale": "NIEW→NIEUW, umlaut removed (modern spelling), regio's apostrophe",
  "rule_id": "§15.6A"
}
```

Every cell has a paper trail: the rule that drove it, the English source, what was there before, what we put in.

### 4.2 The readout rule

No cell was pushed without a full readout:

- Sheet + cell ID
- Speaker (resolved from row key)
- EN (full English source, never truncated)
- NL current (full Dutch, never truncated)
- NL proposed (full Dutch, never truncated)
- Rule (§ reference + one-line rationale)

Tom confirmed before every `--apply`. This discipline caught encoding issues, false positives, and context misreads that automated scripts would have silently pushed through.

### 4.3 Patrick's role

Patrick Claes is the game's senior editor and co-creator. Our editorial passes stacked *on top of* his work, not in competition with it. The policy (§16) was:

- Defer to Patrick on substantive rewrites
- Apply canon consistency rules after his text lands
- When canon and Patrick's proposal conflict — canon wins, and Patrick is informed

In practice this worked well. Several of Patrick's raises confirmed our existing canon; several of his proposals were adopted intact (Topstrateeg, Kameraad Moeder, Vijver van Hiehiehie, Oh jee). A handful were overridden with explanation (Muilebeek → Muilenbeek, De Zatten Ezel Cafe → bare De Zatten Ezel, SCHEP → SCHUP).

## 6. The Big Decisions

These are the locks that shaped the whole corpus. Listed roughly in order of when they were made.

### Place names (§3)

- **Muilenbeek** (not Muilebeek, not Muilegem) — the region. Patrick's sprites were redone to match.
- **Klotegem** — the village (Bumpkin Village). Klotegemmers = the demonym.
- **Technopolis** — the human city (was Mechalen in early corpus). Patrick sweep complete.
- **De Zatten Ezel** — the bar (bare form; Patrick's "Cafe" suffix rejected).
- **de Hoeve** — the farm building (was Boerderij; ~40 drift cells corrected corpus-wide).
- **Erfgoedstal** — the Heritage Stable (was Erfgoedshoeve — wrong word entirely).

### Register and negation (§2, §5)

- **niet** everywhere, no exceptions. No `nie`, no `nie'`. Overrides codex per-character rules for Bad/Nice/Sad/Thirsty.
- **ge/gij chars use u/uw natively** — this is not an exception, it's the full pronoun set. Real exceptions are je/jij chars shifting formal (Sad/Nice/Slow/Grandma/Foal in specific contexts).

### Game-systemic capitalisation (§7.3)

Kudde, Mijn/Mijnen, Mensen, Machines, Goden, Vazallen, Hemelvaart, Heraanstelling, Job/Jobs (context-sensitive), EZELS EERST. These are proper nouns for the world's factions and mechanics — they get capital letters throughout.

### Vocabulary locks (§6)

- **SCHUP** (not SCHEP) — Flemish shovel-noun; reversed an earlier lock.
- **HIER SÉ, KLOOTZAKSKE!!** — Miner Jenny's taunt; Flemish presenter phrase + insult; superseded two earlier forms.
- **Nonkel** everywhere except Smart Ass (who uses Oom).
- **de Plee** (not de Hudo, not Het Privaat, not Sekreet).
- **Job / Jobs** (capped, game-system only; generic post-uprising employment stays lowercase).
- **jansen** — extinct after E6. Two specific cells (gulle / Kleine) replaced it for good.
- **reactie** not `respons` — Anglicism fix.

### Slogans and ritual phrases (§14)

- **EZELS EERST** — the donkey uprising slogan. Supersedes EZELSKRACHT, EZEL MACHT, EZELSKRACHT AAN DE MACHT, EZELKRACHT. 13 drift cells corrected.
- **het Hemelvaarts-zang-der-Ezel-zielen** — the Song of Ascension. Exact form locked with hyphens and capital E in Ezel. 10 cells corrected.
- **Heraanstelling** — the Reassignment mechanic. `herverkoren` and `opnieuw toegewezen` both incorrect; 4 cells fixed.

### The Patrick override table (§16)

| Patrick proposed | Canon final | Outcome |
|---|---|---|
| `Muilebeek` in sprites | `Muilenbeek` | Patrick redoes sprites |
| `De Zatten Ezel Cafe` | bare `De Zatten Ezel` | Patrick reverts |
| `EET MIJN SCHEP OP, KLOOTZAKSKE!!` | `HIER SÉ, KLOOTZAKSKE!!` | Tom's third option |
| "Plan always" | Plan/afspraak nuance kept | EN-driven distinction |
| E1_Stable2F J47 reorder | Tom's canonical form | Grammar + Reassignment translation corrected |

## 10. Craft Notes

### On register specificity

The difference between tussentaal and ABN Standard Dutch is not cosmetic. It's the difference between a character who sounds like they're from Flemish Belgium and one who sounds like a Dutch textbook. Getting it right meant maintaining 37 character voice profiles simultaneously — and those profiles could conflict, drift, and get overridden by Patrick's work between sessions.

The codex solved drift mechanically. The canon solved it institutionally. Together they meant that a decision made in March about Thirsty Ass's article rule would be checked automatically against every Thirsty Ass cell scanned in May.

### On the retcon problem

The project reversed major decisions multiple times. Machien → Machine (120 cells). SCHEP → SCHUP. Boerderij → Hoeve. Nie → niet. Each reversal required finding every previously-pushed cell of the old form and correcting it.

The lesson: lock forms early, lock them hard, and build the archaeology into the document (§18). The retcon table exists precisely so the next session doesn't reverse a reversal.

### On working with a live corpus

Patrick edits the Google Sheets in parallel. Any pull could surface new text that needed our pass, or could show that Patrick had already fixed something we were about to fix, or — occasionally — that Patrick had introduced a new form that conflicted with a canon lock.

The three-way diff script (original baseline → Patrick's version → our version) was the tool that made this workable. It made the conflicts visible without requiring a manual cell-by-cell comparison.

### On the editorial sweep methodology

The §20 protocol — read canon, dry-run, full readout, confirm, push, verify — was slower than a bulk replace. It was also how we caught:
- The em-dash vs colon inconsistency in E2_World_A2 J7 (would have been inconsistent across 4 sign sheets)
- The `Graanopslag` vs `Graanschuur` split in E2 (extra change slipped into the batch)
- The `Helpful Ass ge/gij` flip in E10 (required mid-episode decision about a character the codex didn't have yet)

The slowness was the point. Automated bulk replacement was used for clear-cut cases (Ezel cap, nie→niet, sign typos). Everything with context dependency went through the full readout loop.
